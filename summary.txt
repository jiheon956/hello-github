# 지난 강의 요약
저장소 복제하기 clone - 깃허브에서 생성한 저장소를 로컬에 복사
스테이지에 올리기 add - 로컬에서 작업한 파일을 묶기
저장하기 commit - 스테이지에 올라온 파일을 로컬에 저장 (=save)
업로드하기 push - 원격저장소인 깃허브에 업로드

# 이번 강의 요약
되돌리기 check out - 마지막 저장시점 (커밋)으로 변경사항 되돌리기
코드뭉치 버리기 - 소스트리에서 체크아웃 기능

# 브랜치 개념
브랜치 - 가상의 작업공간으로 기존 내용을 유지한 채 새로 작성할 때 사용
헤드 브랜치 - 브랜치는 한 번에 하나만 작성가능하며 현재 사용 중인 브랜치를 말함
되돌리기 check out - 특정 브랜치 (또는 커밋)으로 돌아가고 싶을 때 사용
소스트리에서 브랜치 체크아웃 - 브랜치 이름 더블 클릭
